AWS Bedrock은 Amazon Web Services가 제공하는 **완전 관리형 생성형 AI 서비스**입니다.

핵심을 간단히 말하면, 여러 AI 회사의 파운데이션 모델(FM)을 **하나의 API로 접근**할 수 있게 해주는 플랫폼이에요. 직접 인프라를 구축하거나 모델을 호스팅할 필요 없이, API 호출만으로 다양한 LLM을 사용할 수 있습니다.좋습니다. 최신 정보까지 포함해서 자세히 설명드리겠습니다.

## AWS Bedrock란?

Amazon Bedrock은 AWS의 완전 관리형(fully managed) 서비스로, 여러 선도적인 AI 회사의 고성능 파운데이션 모델을 하나의 통합 API를 통해 접근할 수 있게 해줍니다. 쉽게 말해, GPU 서버를 직접 프로비저닝하거나 모델을 호스팅할 필요 없이, API 한 번 호출로 세계 최고 수준의 AI 모델을 사용할 수 있는 플랫폼입니다.

---

### 왜 Bedrock이 중요한가?

Scott 입장에서 특히 중요한 포인트는 이겁니다. Bedrock이 대부분의 엔터프라이즈 AI 배포의 기반이 된 이유는 단순합니다 — 기존 AWS 인프라 안에 위치하기 때문입니다. IAM 역할, VPC, CloudWatch, Lambda 함수 등이 S3나 DynamoDB와 동일한 방식으로 Bedrock과 연동됩니다. 이미 AWS를 쓰고 있는 엔터프라이즈 고객 입장에서는 별도의 인프라 변경 없이 AI를 도입할 수 있다는 게 핵심 셀링 포인트입니다.

---

### 제공하는 모델들

2026년 기준 Bedrock은 약 100개에 가까운 서버리스 모델을 제공하며, Anthropic, OpenAI, Meta, Mistral, Google, NVIDIA, MiniMax, Moonshot, Qwen 등 주요 AI 회사들의 모델을 포함합니다.

최근 추가된 모델 예시로는 DeepSeek V3.2, MiniMax M2.1, GLM 4.7, Kimi K2.5, Qwen3 Coder Next 등이 있고, 2026년 3월에는 GLM 5와 MiniMax M2.5도 추가되었습니다.

---

### 핵심 기능 4가지

Bedrock을 단순한 모델 접근 이상으로 강력하게 만드는 네 가지 핵심 요소가 있습니다.

**1. Knowledge Bases (관리형 RAG)**
S3 버킷에 있는 문서를 연결하면 임베딩, 벡터 저장, 검색을 자동으로 처리합니다. 자체 문서 기반 AI 챗봇을 쉽게 만들 수 있습니다.

**2. Agents (에이전트 오케스트레이션)**
모델이 호출할 수 있는 도구(Lambda 함수)를 정의하면, Bedrock이 대화 히스토리와 에이전트 루프를 관리합니다. AgentCore를 통해 에이전트를 안전하게 대규모로 빌드, 배포, 운영할 수 있으며, 어떤 프레임워크와 모델이든 사용 가능합니다.

**3. Guardrails (가드레일)**
유해 콘텐츠의 최대 88%를 차단하고, Automated Reasoning 검사를 통해 환각(hallucination)을 최소화하며 최대 99% 정확도로 올바른 모델 응답을 식별할 수 있습니다. PII 탐지, 주제 제한 등을 애플리케이션 코드가 아닌 플랫폼 레벨에서 처리합니다.

**4. 모델 커스터마이징**
Reinforcement Fine-Tuning(RFT)을 통해 대규모 학습 데이터셋 없이도, 소수의 프롬프트와 피드백만으로 모델을 커스터마이징할 수 있습니다. 파인튜닝 외에도 프롬프트 엔지니어링, Knowledge Base 연동 등 다양한 커스터마이징 옵션이 있습니다.

---

### 비용 최적화 기능

Model Distillation(증류된 모델은 최대 500% 빠르고 75% 저렴), Prompt Caching, Intelligent Prompt Routing(품질 유지하며 비용 최대 30% 절감) 등의 기능을 제공합니다. 토큰당 과금(pay-per-token) 방식이라 초기 투자 없이 시작할 수 있습니다.

---

### 보안 및 컴플라이언스

Bedrock은 고객 데이터를 모델 훈련에 사용하지 않으며, 전송 중/저장 중 데이터 암호화를 제공합니다. ISO, SOC, GDPR, FedRAMP High 등 주요 컴플라이언스 표준을 지원하고 HIPAA도 적용 가능합니다. 국방/금융 같은 규제 산업에서 특히 중요한 포인트입니다.

---

### 실제 활용 사례

Robinhood은 Bedrock을 활용해 일일 토큰 처리량을 5억에서 50억으로 확대하면서도 AI 비용을 80% 절감하고 개발 시간을 절반으로 단축했습니다.

---

### AI 세일즈 관점에서의 포지셔닝

Bedrock은 기본적으로 **AI 모델의 마켓플레이스 + 엔터프라이즈 AI 인프라**입니다. 경쟁 구도를 보면 Azure OpenAI Service(Microsoft), Vertex AI(Google Cloud)와 직접 경쟁하며, "멀티 모델 선택권 + 기존 AWS 인프라 통합"이 차별화 포인트입니다. Redis나 다른 AI 인프라 회사들의 제품이 Bedrock 위에서 동작하거나 연동되는 경우도 많으니, 이 생태계를 이해하고 있으면 면접이나 고객 대화에서 확실히 도움이 될 겁니다.

## AWS Bedrock 아키텍처 계층별 상세 설명

Bedrock을 하나의 "스택"으로 보면, 아래에서 위로 올라가는 계층 구조로 이해할 수 있습니다.

---

### 계층 1: 인프라 레이어 (AWS 기반)

Bedrock의 모든 것은 AWS 글로벌 인프라 위에서 동작합니다. 사용자가 직접 GPU 인스턴스를 관리할 필요가 없다는 게 핵심입니다.

**서버리스 추론(Serverless Inference):** 모델 호출 시 AWS가 자동으로 컴퓨팅 리소스를 할당하고, 사용한 토큰만큼만 과금합니다. EC2나 SageMaker처럼 인스턴스를 프로비저닝할 필요가 없습니다.

**Provisioned Throughput:** 일관된 성능이 필요한 프로덕션 워크로드의 경우, 특정 모델에 대해 전용 처리량을 예약할 수 있습니다. 대규모 트래픽이 예측되는 엔터프라이즈 환경에서 사용합니다.

**Project Mantle:** 새로운 분산 추론 엔진으로, 대규모 ML 모델 서빙을 담당합니다. 새 모델의 온보딩을 간소화하고, 자동화된 용량 관리와 통합 풀을 통해 더 높은 기본 고객 할당량을 제공하며, OpenAI API 스펙과의 호환성도 기본 제공합니다.

**리전 가용성:** 아시아 태평양(뭄바이) 등 주요 리전에서 전체 모델과 핵심 기능을 사용할 수 있어, 데이터가 해당 리전에 머물고 지연 시간이 낮게 유지됩니다. 한국 고객 입장에서는 서울 리전 또는 도쿄 리전 가용 여부가 중요한 셀링 포인트가 됩니다.

---

### 계층 2: 모델 레이어 (Foundation Models)

Bedrock의 가장 기본적인 가치 제안 — 다양한 AI 회사의 모델을 하나의 API로 접근합니다.

**프로프라이어터리 모델:**
Anthropic Claude(3.5 Sonnet, Opus, Haiku 등), Amazon Titan(텍스트, 임베딩, 이미지), AI21 Labs Jamba, Cohere Command/Embed, Stability AI(이미지 생성) 등이 포함됩니다.

**오픈 웨이트 모델:**
Google, MiniMax, Mistral AI, Moonshot AI, NVIDIA, OpenAI, Qwen 등의 모델을 포함하며, 2026년 기준 약 60개에서 거의 100개 모델로 확대되었습니다. 텍스트, 비전, 오디오, 안전성, 코드 워크로드를 아우르는 멀티모달 플랫폼으로 진화했습니다.

**모델 선택의 유연성:**
통합 API를 통해 모델을 평가, 전환, 채택할 수 있으며 애플리케이션을 다시 작성하거나 인프라를 변경할 필요가 없습니다. 이것이 vendor lock-in 없이 최적의 모델을 선택할 수 있다는 Bedrock의 핵심 메시지입니다.

**모델 평가 도구:**
Bedrock 콘솔 내에서 여러 모델을 나란히 비교하고 벤치마크할 수 있습니다. 성능, 비용, 정확도 기준으로 자사 유즈케이스에 맞는 최적 모델을 데이터 기반으로 선택할 수 있게 해줍니다.

---

### 계층 3: 커스터마이징 레이어

범용 모델을 "우리 회사 모델"로 만드는 계층입니다.

**프롬프트 엔지니어링:**
가장 가볍고 빠른 커스터마이징. 시스템 프롬프트, few-shot 예제 등을 통해 모델 행동을 조정합니다. 비용이 거의 들지 않습니다.

**Knowledge Bases (RAG):**
S3 버킷에 있는 문서를 연결하면 Bedrock이 임베딩, 벡터 저장, 검색을 자동으로 처리합니다. Amazon OpenSearch Serverless, Amazon Aurora, Pinecone 등 다양한 벡터 스토어와 연동됩니다. 회사 내부 문서, 제품 매뉴얼, FAQ 등을 모델에게 "가르치는" 가장 실용적인 방법입니다.

**Bedrock Data Automation:**
비정형 데이터(PDF, 이미지, 비디오 등)를 자동으로 구조화된 데이터로 변환해주는 기능입니다. Knowledge Base에 넣기 전 데이터 전처리를 자동화합니다.

**Fine-Tuning:**
전통적인 방식으로, 학습 데이터를 제공해 모델 가중치를 직접 조정합니다. 도메인 특화 용어나 스타일이 필요할 때 사용합니다.

**Reinforcement Fine-Tuning (RFT):**
대량의 레이블링된 데이터 없이도 소수의 프롬프트에서 여러 가능한 응답에 대한 피드백을 통해 모델이 학습할 수 있습니다. 규칙 기반 그레이더나 AI 기반 심사위원을 사용한 보상 함수를 정의하고, 학습 중 중간 체크포인트에 접근하여 평가, 디버깅, 최적 모델 선택이 가능합니다.

**Model Distillation (모델 증류):**
증류된 모델은 최대 500% 빠르고 75% 저렴하면서 정확도에 미치는 영향은 최소화됩니다. 큰 모델의 지식을 작은 모델에 전이시켜 프로덕션 비용을 획기적으로 줄이는 기법입니다.

---

### 계층 4: 오케스트레이션 레이어 (Agents & AgentCore)

모델이 단순히 텍스트를 생성하는 것을 넘어, **행동하는 AI**로 만드는 계층입니다.

**Bedrock Agents:**
가이드 방식의 에이전트 빌딩 도구입니다. 자연어로 에이전트의 역할을 정의하고, 호출할 수 있는 도구(API/Lambda)를 연결하면, Bedrock이 ReAct 패턴으로 자동 오케스트레이션합니다. 예를 들어 "고객 주문 상태를 조회하고, 환불이 필요하면 처리해줘"라는 복잡한 워크플로우를 하나의 에이전트로 처리할 수 있습니다.

**AgentCore:**
어떤 프레임워크와 모델이든 사용해서 에이전트를 빌드, 배포, 운영할 수 있는 에이전틱 플랫폼입니다. 인프라 관리 없이 도구와 데이터에 걸쳐 적절한 권한과 제어로 에이전트가 행동을 수행할 수 있게 합니다. AgentCore는 여러 하위 서비스로 구성됩니다.

- **Runtime:** 안전한 서버리스 배포 환경
- **Gateway:** 통합 도구 접근 및 연결 관리
- **Memory:** 세션 간 지능형 컨텍스트 유지 — 에이전트가 이전 대화 맥락을 기억
- **Identity:** AWS 및 서드파티 서비스 간 인증 처리
- **Browser & Code Interpreter:** 에이전트가 웹 브라우징이나 코드 실행 가능
- **Observability:** 모니터링과 디버깅

**멀티 에이전트 조정:**
코디네이터 에이전트가 전문화된 서브 에이전트에게 도구 호출을 통해 작업을 위임하며, 각 에이전트는 격리된 컨텍스트 윈도우를 가집니다. 예를 들어 "리서치 에이전트"가 정보를 수집하고, "분석 에이전트"가 판단하고, "실행 에이전트"가 조치를 취하는 파이프라인을 구성할 수 있습니다.

2026년 3월에는 AgentCore의 정책 제어(Policy Controls)가 GA되어, 에이전트가 취할 수 있는 행동을 에이전트의 추론 루프 바깥에서 정밀하게 제어할 수 있게 되었습니다. Stateful MCP 서버 지원과 메모리 스트리밍 알림도 확장되었습니다.

---

### 계층 5: 안전성 및 거버넌스 레이어 (Guardrails)

엔터프라이즈에서 AI를 프로덕션에 배포할 때 가장 큰 걱정은 "모델이 이상한 말을 하면 어쩌지?"입니다. Guardrails가 이것을 처리합니다.

**콘텐츠 필터링:** 유해하거나 부적절한 콘텐츠를 입력/출력 양쪽에서 차단합니다. 카테고리별로 엄격도를 조절할 수 있습니다.

**Denied Topics:** 특정 주제(예: 경쟁사 정보 제공, 투자 조언 등)에 대해 모델이 응답하지 않도록 설정합니다.

**PII 탐지 및 마스킹:** 개인정보(이름, 주민번호, 카드번호 등)를 자동으로 탐지하고 마스킹합니다.

**Automated Reasoning:** 자동화된 추론 검사를 통해 환각을 최소화하며, 최대 99% 정확도로 올바른 모델 응답을 식별할 수 있습니다.

**Structured Outputs:** 모델이 일관되고 스키마를 준수하는 기계 판독 가능한 응답을 반환하도록 보장합니다. JSON 스키마를 정의하거나 엄격한 도구 정의를 사용할 수 있어, 사소한 포맷 오류로 다운스트림 시스템이 깨지는 것을 방지합니다.

---

### 계층 6: 비용 최적화 레이어

**Pay-per-Token:** 기본 과금 모델. 사용한 입력/출력 토큰에 대해서만 비용을 지불합니다.

**Prompt Caching:** 반복되는 시스템 프롬프트나 컨텍스트를 캐싱하여 비용을 절감합니다.

**Intelligent Prompt Routing:** 품질을 유지하면서 비용을 최대 30% 절감할 수 있습니다. 간단한 질문은 저렴한 모델로, 복잡한 질문은 고성능 모델로 자동 라우팅합니다.

**Batch Processing:** 실시간 응답이 필요 없는 대량 처리(예: 1만 건의 문서 요약)를 할인된 가격으로 처리합니다.

---

### 계층 7: 관측성 및 운영 레이어

**CloudWatch 통합:** 지연 시간 분포, 토큰 비용, 도구 호출 빈도, 에이전트별 에러율 등을 모니터링합니다.

**로깅 및 감사:** 모든 API 호출과 모델 입출력을 S3, CloudWatch Logs에 기록하여 컴플라이언스 감사에 대응할 수 있습니다.

**모델 호출 로깅:** 어떤 모델이 언제, 어떤 입력에 대해 어떤 응답을 했는지 전부 추적 가능합니다. 규제 산업(금융, 국방, 의료)에서 필수적인 기능입니다.

### 세일즈 관점에서의 시사점

이 계층 구조를 이해하면 고객 대화에서 강력합니다. 고객의 성숙도에 따라 진입점이 다르거든요. 처음 AI를 도입하는 고객은 계층 2(모델 접근)부터 시작하고, 이미 PoC를 진행한 고객은 계층 3~4(커스터마이징/에이전트)에 관심이 있고, 프로덕션 배포를 앞둔 고객은 계층 5~7(거버넌스/비용/운영)이 결정적인 구매 요인이 됩니다.
